자바스크립트 예외처리

개요

프로그램의 실행을 중단시키는 요인을 에러 또는 예외라고 함.

에러(error) : 개발자가 조치할 수 없는 시스템 상의 오류.
 ㄴ 네트워크 오류, 전원 차단으로 인한 오류 등

예외(exception) : 개발자가 조치할 수 있는 시스템 상의 오류.
 ㄴ 문법 오류, 논리 오류, 실행 오류 등

자바스크립트의 에외는 try - catch 문을 통해 관리 가능.

try {

예외가 발생할만한 코드들

} catch(객체) {

예외 처리 코드

}

오류 정보 확인

Error 객체의 name, message, stack 속성으로 확인 가능함.

유형별 예외처리

오류 유형별로 메세지 처리를 상이하게 하고 싶다면 instanceof 연산자를 사용하면 됨.

ReferenceError : 변수 선언과 관련된 오류 발생 시 출력됨.

RangeError : 범위와 관련된 오류 발생 시 출력됨.

SyntaxError : 문법과 관련된 오류 발생 시 출력됨.

TypeError : 변수나 객체의 자료형과 관련된 오류 발생 시 출력됨.

인위적으로 오류발생시키기

특정 코드에 의해 발생되는 오류 외에도 로직이 정상적으로 수행되는 특정조건을 만족하지 못했을때 의도적으로 오류를 발생시켜 catch 블럭에 의해 예외처리를 해야할 경우가 있음

이때 'throw 오류객체명' 구문으로 오류를 발생시킬수 있음
